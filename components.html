Parameters:

  • id (AutoField) – Primary key: ID

  • overlay_segments (JSONField) – Overlay segments. The schema that defines how categories of values in the overlay images are differentiated. Example usage: [{“name”: “background”, “visible”: true, “voxel_value”: 0},{“name”: “tissue”, “visible”: true, “voxel_value”: 1}]. If a categorical overlay is shown, it is possible to show toggles to change the visibility of the different overlay categories. To do so, configure the categories that should be displayed. For example: [{“name”: “Level 0”, “visible”: false, “voxel_value”: 0].

class grandchallenge.components.models.InterfaceKinds(value)[source]¶

Interface kind sets.

Interface kinds that are images:

  - Image
  - Displacement field
  - Segmentation
  - Heat Map
  - DICOM Image Set


Interface kinds that are files:

  - BIOM format
  - MP4 file
  - SQREG file
  - Thumbnail png
  - Newick tree-format file
  - Thumbnail jpg
  - PDF file
  - ZIP file
  - OBJ file
  - CSV file


Interface kinds that are json serializable:

  - Ellipse
  - Integer
  - Multiple distance measurements
  - Chart
  - Line
  - Point
  - Multiple angles
  - Anything
  - Float
  - Multiple 2D bounding boxes
  - Polygon
  - Multiple polygons
  - Angle
  - Multiple lines
  - 2D bounding box
  - String
  - Bool
  - Multiple choice
  - Affine transform registration
  - Multiple three-point angles
  - Choice
  - Three-point angle
  - Multiple ellipses
  - Multiple points
  - Distance measurement
